# Dink Card

Dink Card is a React + Node platform for verified users to add service funds, request virtual cards, manage cards, and submit support/KYC requests.

## Production Recommendation

Use Render for the full app at `https://dinkcard.et`.

This codebase currently uses:

- Express server sessions
- SQLite database file
- Local upload storage
- Provider webhooks

Render supports this flow when you attach a persistent disk. Vercel is excellent for the frontend, but Vercel serverless storage is ephemeral, so do not run this SQLite/upload backend on Vercel unless you first migrate the database to managed Postgres and uploads to object storage.

## Local Development

```bash
npm install
npm run dev
```

- Frontend: `http://localhost:5173`
- API: `http://localhost:3001`
- Superadmin: `http://localhost:5173/superadmin`

## Required Secrets

Generate strong local values:

```bash
npm run secrets
```

Set these in production hosting:

- `JWT_SECRET`: generated by you.
- `COOKIE_SECRET`: generated by you.
- `CARD_DATA_ENCRYPTION_KEY`: generated by you.
- `SUPERADMIN_PASSWORD`: create a strong owner password.
- `SUPERADMIN_EMAIL`: owner email.
- `CHAPA_PUBLIC_KEY`: from Chapa.
- `VITE_CHAPA_PUBLIC_KEY`: same Chapa public key for frontend build.
- `CHAPA_SECRET_KEY`: from Chapa.
- `CHAPA_WEBHOOK_SECRET`: webhook hash/secret configured in Chapa.
- `BITNOB_CLIENT_ID`: from Bitnob.
- `BITNOB_CLIENT_SECRET`: from Bitnob.
- `BITNOB_WEBHOOK_SECRET`: Bitnob webhook signing secret. If Bitnob does not expose a separate test webhook secret, use the configured signing secret they provide for webhooks.

Never put live secrets in GitHub.

## Render Full-Stack Deploy

Recommended setup for `dinkcard.et`.

1. Push this repository to GitHub.
2. In Render, create a new Blueprint from the repo, or create a Web Service manually.
3. If using Blueprint, Render reads `render.yaml`.
4. If creating manually:
   - Runtime: `Node`
   - Build command: `npm ci --include=dev && npm run build`
   - Start command: `npm run start`
   - Health check path: `/api/health`
   - Add persistent disk mounted at `/var/data`
5. Add production environment variables:
   - `NODE_ENV=production`
   - `APP_URL=https://dinkcard.et`
   - `API_URL=https://dinkcard.et`
   - `VITE_API_BASE_URL=`
   - `DATABASE_PATH=/var/data/dinkcard.db`
   - `UPLOAD_DIR=/var/data/uploads`
   - `CHAPA_CALLBACK_URL=https://dinkcard.et/api/payments/chapa/callback`
   - `CHAPA_RETURN_URL=https://dinkcard.et/add-money`
   - `BITNOB_WEBHOOK_URL=https://dinkcard.et/api/webhooks/bitnob`
6. Add all secrets from the Required Secrets section.
7. Deploy.
8. Add custom domain `dinkcard.et` in Render and update your DNS records as Render instructs.
9. In Chapa, set callback/webhook URLs to the `https://dinkcard.et` URLs.
10. In Bitnob, set webhook URL to `https://dinkcard.et/api/webhooks/bitnob`.

## Optional Vercel Frontend + Render API

Only use this if you want Vercel to host the React frontend and Render to host the API.

1. Deploy the backend on Render with a custom domain like `https://api.dinkcard.et`.
2. In Render set:
   - `APP_URL=https://dinkcard.et`
   - `API_URL=https://api.dinkcard.et`
   - `CHAPA_CALLBACK_URL=https://api.dinkcard.et/api/payments/chapa/callback`
   - `CHAPA_RETURN_URL=https://dinkcard.et/add-money`
   - `BITNOB_WEBHOOK_URL=https://api.dinkcard.et/api/webhooks/bitnob`
3. In Vercel import the repo.
4. Set framework preset to Vite.
5. Build command: `npm run build`
6. Output directory: `dist`
7. Set frontend env:
   - `VITE_API_BASE_URL=https://api.dinkcard.et`
   - `VITE_CHAPA_PUBLIC_KEY=your_chapa_public_key`
8. Add custom domain `dinkcard.et` in Vercel and set DNS as Vercel instructs.

Do not deploy the current backend/database/uploads to Vercel serverless as-is.

## Production Safety Checklist

- Change the superadmin password before launch.
- Use live Chapa and Bitnob keys only in production env variables.
- Keep `NODE_ENV=production` in production.
- Keep `DATABASE_PATH` and `UPLOAD_DIR` on a persistent disk.
- Confirm `/api/health` returns `{"ok":true}` after deploy.
- Confirm signup requires Terms acceptance.
- Confirm KYC upload, KYC submit, admin approval, add funds, and card request flows.
- Confirm Chapa verifies from backend before crediting balance.
- Confirm Bitnob webhook rejects bad signatures.
- Keep regular backups of `/var/data/dinkcard.db` and `/var/data/uploads`.
